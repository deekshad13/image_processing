Root Cause: Baseline DINOv2 sees "photos of leaves" — not "disease symptoms." It compares general visual features (leaf shape, green color, background blur). All leaf symptoms look similar to it because they're all... leaves. The model has zero agricultural knowledge.

Fix 1: Deploy the Fine-Tuned Model — Highest Impact

The intern already trained a fine-tuned model that showed +27% precision improvement in evaluation. But the weights file (fine_tuned_model.pt) was trained on Google Colab and never deployed. The API is running on raw baseline DINOv2 — all that training work is sitting unused.

What fine-tuning does

The CropSimilarityModel adds a projection head (384→256→128 dims) on top of DINOv2 and trains with triplet loss:

  • Anchor: A Galls image
  • Positive: A different Galls image (should be close in embedding space)
  • Negative: A Leaf Color Change image (should be far apart)
  • The loss function literally teaches: "Galls images should cluster together, away from non-Galls images"

Action

  1. Retrieve fine_tuned_model.pt from the Google Colab session where it was trained (check Colab file browser or Google Drive)
  2. Place it at data/embeddings/fine_tuned_model.pt
  3. Run python scripts/build_gallery.py to rebuild embeddings with the fine-tuned model
  4. Restart the server — it auto-detects and loads the fine-tuned model

If the weights are lost, re-train locally or on Colab:

python scripts/train.py   # ~20 min on GPU, ~2 hours on CPU

Expected improvement: The intern's evaluation showed Precision@5 going from 0.71 → 0.90. The weak leaf symptoms and Stem Cracking should see the biggest gains since fine-tuning specifically teaches the model to separate them.

Fix 2: Better Training Loss — ArcFace Instead of Triplet — High Impact

Triplet loss (current) is okay but outdated. It only learns from 3 images at a time (anchor, positive, negative), and with random sampling, most triplets are "easy" — the model already knows Galls ≠ Fruit, so those triplets contribute zero learning.

Why ArcFace is better for verification

AspectTriplet Loss (Current)ArcFace Loss (Recommended)
Learning signal3 images per stepEntire batch (16+ images) per step
Hard casesRandom sampling — often picks easy negativesAutomatically focuses on hardest distinctions
Task fitGeneral metric learningSpecifically designed for verification ("is this the same?")
Used inGeneral similarityFace verification (exactly analogous to our symptom verification)
ConvergenceSlow, requires careful margin tuningFaster, more stable

Implementation

pip install pytorch-metric-learning

Replace nn.TripletMarginLoss in fine_tune.py with:

from pytorch_metric_learning import losses, miners

loss_fn = losses.ArcFaceLoss(
    num_classes=10,      # 10 symptom classes
    embedding_size=128,  # projection dim
)
miner = miners.MultiSimilarityMiner()  # hard-negative mining

# Training loop:
embeddings = model(batch_images)
hard_pairs = miner(embeddings, labels)
loss = loss_fn(embeddings, labels, hard_pairs)

Expected improvement: 15-30% better separation between similar symptoms (leaf classes). This is the single biggest algorithmic change available.

Data sources

  • Field collection — Real farmer photos from the EywaFarm platform. Most valuable because they match actual usage conditions.
  • Web scraping — Agricultural extension sites, university disease databases.

Quick win: Data augmentation

Without collecting a single new image, we can multiply effective dataset size 5-10x by generating variations during training:

augment_transform = transforms.Compose([
    transforms.RandomResizedCrop(224, scale=(0.6, 1.0)),
    transforms.RandomHorizontalFlip(),
    transforms.RandomVerticalFlip(),
    transforms.ColorJitter(brightness=0.3, contrast=0.3, saturation=0.2),
    transforms.RandomRotation(30),
    transforms.ToTensor(),
    transforms.Normalize(mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225]),
])

This teaches the model that a Galls image is still Galls when rotated, zoomed, or under different lighting.

Fix 4: Smarter Comparison — Center-Weighted Similarity — Medium Impact

Currently, the entire image (including background, fingers, soil) contributes equally to the embedding. For a macro crop photo, 40-60% of the image might be irrelevant background. The disease symptom is in the center.

Approach: Multi-crop embedding

Instead of one embedding per image, extract 3:

  1. Full image — overall context
  2. Center crop (60%) — focuses on the subject
  3. Tight center crop (40%) — focuses on the symptom detail

Weighted average: 20% full + 40% center + 40% tight. This makes the comparison focus on the disease, not the background.
