What Rootstalk Does

Rootstalk helps farmers verify crop disease symptoms. When a farmer sees something wrong with their crop and navigates to a symptom page (e.g., "Galls"), they can upload a photo and get an instant similarity score telling them how closely their crop matches that symptom.

This is not a diagnosis engine. It's a decision support tool that guides the farmer's judgment with data: "Your image is 73% similar to Galls — Monitor and recheck."

The Problem We Solved

Before (Incorrect Approach)

The initial prototype searched across ALL symptoms and returned "Your image is most similar to Leaf Rust." This is a disease identification tool — not what was needed.

After (Corrected Approach)

Rootstalk now performs single-symptom verification. The farmer is already on a symptom page. They want to know: "Does my plant match THIS symptom?" — a fundamentally different, more useful question.

How the Technology Works

Step 1: Image → Embedding Vector

A vision AI model (like DINOv2) converts any image into a high-dimensional number vector (e.g., 384 numbers). Images that look similar produce similar vectors. This is the core insight that makes image comparison possible without pixel-by-pixel matching.

Step 2: Gallery of Reference Vectors

All reference symptom images are pre-converted to vectors and stored. When a farmer uploads a photo, their image is also converted to a vector in the same space.

Step 3: Cosine Similarity

The angle between the farmer's image vector and each reference vector is measured. Smaller angle = higher similarity. The top-N most similar reference vectors are averaged to produce a robust percentage score.

Step 4: Recommendation

Based on configurable thresholds:

SimilarityRecommendationMeaning
≥80%Take ActionHigh confidence — consult an expert
60–79%MonitorModerate — observe and recheck
<60%Low ConcernLikely not a match

Multi-Engine Model Comparison

Different AI models "see" images differently. To find the best model for crop disease verification, Rootstalk supports 6 different vision AI engines that can be tested side by side on the same image.

EngineFamilyParametersEmbedding DimBest For
DINOv2 ViT-S/14 DINOv2 21M384 Best general visual similarity. Lightweight. Current default.
DINOv2 ViT-B/14 DINOv2 86M768 Fine-grained distinctions. Better for subtle disease differences.
CLIP ViT-B/32 CLIP 151M512 Semantic understanding. Knows what things "are," not just how they look.
CLIP ViT-L/14 CLIP 428M768 Richest semantic features. Best for conceptual matching.
ConvNeXt Base CNN 89M1024 Local texture patterns. Good for texture-based diseases (spots, rust).
ResNet-50 CNN 25M2048 Classic baseline. Fastest inference. Reference benchmark.

Why Multiple Engines?

  • DINOv2 (Meta) — Self-supervised learning. Understands visual structure without needing labels. Best for "does this look similar?"
  • CLIP (OpenAI) — Trained on 2 billion image-text pairs. Understands what things are. Better at "is this a gall?" conceptually.
  • ConvNeXt / ResNet — Traditional CNNs. Excellent at local texture patterns (spots, cracks, holes). Fast and lightweight.

By testing the same image across all engines, we can determine which model family works best for each type of crop symptom — data-driven model selection rather than guesswork.

System Features

Image Validation

Before processing, every uploaded image is checked for:

  • Format: JPG/PNG only
  • Corruption: OpenCV decode check
  • Brightness: Rejects too dark (<30) or too bright (>240) images
  • Blur: Laplacian variance check — rejects motion-blurred photos
  • Size: Configurable max file size

All thresholds are configurable at runtime via the Settings panel or API — no server restart needed.

Reference Image Gallery

When verifying a symptom, the farmer sees a grid of reference images showing what that symptom looks like. They can click any reference to view it full-size and visually compare with their own photo.

Runtime Configuration

All critical parameters are adjustable without restarting:

  • Blur threshold, brightness range, file size limit
  • Take Action / Monitor thresholds
  • Number of reference images to average (Top-N)
